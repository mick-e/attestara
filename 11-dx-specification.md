# Attestara — Developer Experience (DX) Specification
## v0.1

---

## 1. DX Philosophy

A protocol is only as good as the experience of building with it. The Attestara SDK must meet one standard: **a developer with no prior ZK knowledge should be able to run a complete agent-to-agent negotiation with verified credentials and an on-chain commitment record in under 30 minutes.**

Everything in this specification flows from that standard. If a feature adds complexity without reducing this time or increasing capability, it is deferred.

---

## 2. "Hello World" Negotiation — Target Experience

This is the experience every developer should have on first contact with Attestara:

```bash
# Install
npm install @attestara/sdk

# Setup (one-time)
npx attestara init
# > Creates .attestara config
# > Generates agent DID (or imports existing)
# > Fetches test credentials from Attestara testnet
# > Deploys test verifier contract (optional, uses shared testnet by default)

# ✅ Ready in ~2 minutes
```

```typescript
// negotiate.ts — complete working example in 30 lines

import { AttestaraClient, mockAgent } from '@attestara/sdk'

// Agent A (buyer) — your agent
const buyer = new AttestaraClient({ 
  config: '.attestara'  // loads from init
})

// Agent B (seller) — mock for testing
const seller = mockAgent({
  did: 'did:ethr:arb-goerli:0xSeller...',
  maxPrice: 600000, 
  minDelivery: 30
})

// Initiate session — credentials and proofs handled automatically
const session = await buyer.negotiate({
  counterparty: seller.did,
  subject: 'test.procurement',
  offer: { value: 400000, currency: 'EUR', deliveryDays: 60 }
})

// Receive counter-offers
session.on('counter', (turn) => {
  console.log('Seller offered:', turn.terms)
  // Accept if within budget
  if (turn.terms.value <= 480000) {
    session.accept(turn)
  } else {
    session.counter({ value: turn.terms.value - 10000, deliveryDays: 50 })
  }
})

// On agreement
session.on('agreement', async (deal) => {
  console.log('Deal agreed:', deal.terms)
  const commitment = await session.commit(deal)
  console.log('On-chain commitment:', commitment.txHash)
  // ✅ Verifiable commitment record on Arbitrum Goerli
})

await session.start()
```

## 3. SDK Architecture

### 3.1 Package Structure

```
@attestara/sdk
├── core
│   ├── AttestaraClient     Main entry point
│   ├── NegotiationSession   Session lifecycle management
│   └── CommitmentManager    On-chain commitment interactions
├── credentials
│   ├── CredentialManager    VC issuance and verification
│   ├── DIDManager           DID creation and resolution
│   └── AuthorityCredential  Typed credential helpers
├── proofs
│   ├── ProofGenerator       ZK proof generation (local or remote)
│   ├── ProofVerifier        Proof verification
│   └── circuits/            Compiled circuit artifacts
├── contracts
│   ├── CommitmentContract   Typed contract interactions
│   ├── AgentRegistry        Registry read/write
│   └── abis/                Contract ABIs
├── testing
│   ├── MockAgent            Test counterparty
│   ├── TestCredentials      Pre-issued test credentials
│   └── LocalChain           In-memory chain for unit tests
└── types
    └── index.ts             All exported TypeScript types
```

### 3.2 Configuration

`.attestara` config file (generated by `npx attestara init`):

```json
{
  "version": "1",
  "agent": {
    "did": "did:ethr:arb1:0xYourAgentAddress",
    "keyFile": "./keys/agent.key",  
    "credentialFile": "./credentials/authority.vc.json"
  },
  "network": {
    "chain": "arbitrum-one",
    "rpcUrl": "https://arb1.alchemy.com/v2/${ALCHEMY_API_KEY}",
    "contracts": {
      "commitmentContract": "0x...",
      "agentRegistry": "0x...",
      "credentialRegistry": "0x..."
    }
  },
  "prover": {
    "mode": "local",  // "local" | "remote" | "url"
    "remoteUrl": "https://prover.attestara.io"
  }
}
```

### 3.3 Key Abstractions

**AttestaraClient** — The main class. Handles identity, credentials, and session orchestration.

```typescript
class AttestaraClient {
  // Configuration
  constructor(options: AttestaraOptions)
  
  // Session management
  async initiateSession(params: SessionParams): Promise<NegotiationSession>
  async receiveSession(proposal: SessionProposal): Promise<NegotiationSession>
  
  // Credential management  
  async loadCredential(credential: AuthorityCredential): Promise<void>
  async issueCredential(params: CredentialParams): Promise<AuthorityCredential>  // Principal use
  async verifyCredential(credential: AuthorityCredential): Promise<VerificationResult>
  
  // Identity
  get did(): string
  async registerAgent(): Promise<TransactionReceipt>
}
```

**NegotiationSession** — Manages the lifecycle of a single negotiation.

```typescript
class NegotiationSession {
  // Send turns (proofs generated automatically)
  async sendOffer(terms: NegotiationTerms): Promise<void>
  async sendCounter(terms: NegotiationTerms): Promise<void>
  async accept(turn: NegotiationTurn): Promise<void>
  async reject(reason?: string): Promise<void>
  
  // Events
  on(event: 'offer', handler: (turn: NegotiationTurn) => void): this
  on(event: 'counter', handler: (turn: NegotiationTurn) => void): this
  on(event: 'agreement', handler: (deal: FinalAgreement) => void): this
  on(event: 'escalation', handler: (req: EscalationRequest) => void): this
  
  // Commitment
  async commit(agreement: FinalAgreement): Promise<CommitmentRecord>
  
  // Audit
  async exportAuditTrail(): Promise<AuditTrail>
  get sessionId(): string
  get status(): SessionStatus
}
```

## 4. Framework Integration Adapters

### 4.1 LangChain (Python)

```python
from attestara import AttestaraTool, AttestaraCredential

# Load credential
credential = AttestaraCredential.from_file("authority_credential.json")

# Create Attestara tool
attestara = AttestaraTool(
    credential=credential,
    network="arbitrum-one",
    description="Negotiate and commit to B2B agreements with verified authority"
)

# Use in a LangChain agent
from langchain.agents import initialize_agent, AgentType
from langchain_openai import ChatOpenAI

agent = initialize_agent(
    tools=[attestara, ...other_tools],
    llm=ChatOpenAI(model="gpt-4"),
    agent=AgentType.OPENAI_FUNCTIONS,
    verbose=True
)

result = agent.run(
    "Negotiate a software development contract with SupplierAgent "
    "did:ethr:arb1:0xSupplier... for €300,000-€450,000 over 6 months"
)
```

### 4.2 AutoGen (Python)

```python
from autogen import AssistantAgent, UserProxyAgent
```

### 4.3 Google A2A Integration

Attestara credentials can be embedded in A2A Agent Cards:

```json
{

  "description": "Handles B2B procurement negotiations",
  "url": "https://agents.acme.com/procurement",
  "version": "1.0",
  "capabilities": {
    "streaming": true,
    "pushNotifications": false
  },
  "attestara": {
    "did": "did:ethr:arb1:0xProcurementAgent",

    "network": "arbitrum-one"
  }

```

Counterparties discovering this Agent Card know it supports Attestara trust verification and can initiate a verified session.

## 5. CLI Tools

```bash
# Agent management
attestara agent create                    # Create new agent DID + keys
attestara agent register                  # Register on-chain
attestara agent status                    # Show DID + credential status
attestara agent export --format did-doc   # Export DID document

# Credential management  
attestara credential issue \
  --agent did:ethr:arb1:0xAgent \
  --max-value 500000 \
  --currency EUR \
  --domain procurement.contracts \
  --expires 90d                            # Issue authority credential

attestara credential verify ./cred.json   # Verify a credential
attestara credential revoke ./cred.json   # Revoke a credential

# Session management
attestara session list                    # List active sessions
attestara session audit <sessionId>       # Export audit trail
attestara session status <sessionId>      # Check session status

# Commitment management
attestara commitment list                 # List commitments
attestara commitment get <commitmentId>   # Get commitment details
attestara commitment verify <commitmentId> # Verify on-chain record

# Development utilities
attestara mock-agent                      # Start a local mock negotiation counterparty
attestara testnet faucet                  # Get test ETH on Arbitrum Goerli
attestara circuit benchmark               # Run ZK proof benchmarks
```

## 6. Testing Infrastructure

### 6.1 Unit Testing

```typescript
import { AttestaraTestEnvironment } from '@attestara/sdk/testing'

describe('Negotiation session', () => {
  let env: AttestaraTestEnvironment
  
  beforeAll(async () => {
    // Spins up local chain fork + deploys test contracts
    env = await AttestaraTestEnvironment.create({
      mode: 'local',
      withTestCredentials: true
    })
  })
  
  it('should complete a 3-turn negotiation and commit', async () => {
    const buyer = env.createAgent({ maxBudget: 500000 })
    const seller = env.createMockAgent({ minPrice: 300000 })
    
    const session = await buyer.negotiate({
      counterparty: seller.did,
      offer: { value: 380000, currency: 'EUR' }
    })
    
    // Mock agent auto-responds based on its parameters
    const commitment = await session.waitForCommitment()
    
    expect(commitment.status).toBe('ACTIVE')
    expect(commitment.txHash).toBeTruthy()
  })
})
```

### 6.2 Integration Testing

The SDK includes a testnet integration test suite that runs against Arbitrum Goerli with real deployed contracts and real ZK proofs. These tests run in CI on every PR.

### 6.3 Mock Agent

The `MockAgent` class simulates a counterparty with configurable negotiation behaviour for testing:

```typescript
const seller = new MockAgent({
  did: 'did:ethr:arb-goerli:0xTest...',
  credential: testCredentials.seller,
  strategy: 'concede-slowly',  // built-in strategies
  minAcceptablePrice: 400000,
  maxDeliveryDays: 60
})
```

Built-in strategies: `concede-slowly`, `concede-quickly`, `hold-firm`, `random`, `always-accept`.

## 8. Developer Feedback Loop

**Before launch, the DX must be validated by:**
- 3 developers who have never seen Attestara (cold usability test — target: hello world in <30 min)
- 2 enterprise developers integrating with LangChain or A2A
- 1 developer with ZK experience (validating the circuit documentation)

**Feedback channels:**
- Discord #developer-feedback channel
- GitHub Issues (feature requests and bugs)
- Monthly developer survey in first 6 months
